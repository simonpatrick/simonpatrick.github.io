书 -Books

有些书对于一个测试来说是很有用处的，比如：

  • Lessons Learned in Software Testing: A Context Driven Approach by Cem Kaner, James Bach, and Bret Pettichord
  • Exploratory Software Testing: Tips, Tricks, Tours, and Techniques to Guide Test Design by James A. Whittaker
  • Agile Testing: A Practical Guide for Testers and Agile Teams by Lisa Crispin and Janet Gregory
  • How to Break Web Software: Functional and Security Testing of Web Applications and Web Services by Mike Andrews and James A. Whittaker
  • How We Test Software at Microsoft (PRO-best Practices) by Alan Page, Ken Johnston and Bj Rollison
  • The Art of Software Security Testing: Identifying Software Security Flaws by Chris Wysopal
  • The Web Application Hacker’s Handbook: Discovering and Exploiting Security Flaws by Dafydd Stuttard and Marcus Pinto
  • TMap Next by Tim Koomen, Leo van de Aalst, Bart Broekman and Michiel Vroon

几个问题给一个测试

  • How to improve the software testing craft?
  • What is the most valuable “tool” you use to aid your testing?
  • What’s the most “impressive” bug you’ve ever found?
  • What’s the biggest mistake you’ve ever made when testing?
  • What’s the biggest challenge facing Testing currently?
  • What do you think will be the biggest challenge over the next 2-3 years?
  • What are your views on Testing Certification Schemes?
  • Why do you love software testing?

以上问题仅供大家思考，没有什么标准答案
